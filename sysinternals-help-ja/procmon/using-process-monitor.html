Process Monitor 日本語解説 - Process Monitor の使い方

Sysinternals 日本語解説 > Process Monitor > Process Monitor の使い方

Process Monitor を実行するにはローカル管理者グループのメンバーである必要があります。Process Monitor を起動すると、すぐにファイルシステム、レジストリ、プロセスへの操作に対する監視が開始されます。

  • File System
    Process Monitor は Windows ファイルシステムへのすべてのアクティビティを表示します。これにはローカルストレージとリモートファイルシステムへのアクセスを含みます。Process Monitor は新しいファイルシステムデバイスの追加を自動的に検出し、それらを監視します。すべてのファイルシステムパスは、ファイルシステムへの操作を実行したユーザーセッションに関連付けられて表示されます。例えば、ユーザー A が共有フォルダーをドライブ文字 Z: としてマウントした場合、その共有フォルダーへのアクセスは、Process Monitor では Z ドライブへのものとして表示されます。

    ファイルシステムへの操作を非表示にするにはツールバーの Show File System Activity ボタンの選択を解除し、ファイルシステムへの操作を再表示するにはそのボタンを押します。

  • Registry
    Process Monitor はレジストリに対するすべての操作を記録します。レジストリパスの表示にはレジストリルートキーに慣習的な略語を使います (例えば HKEY_LOCAL_MACHINE は HKLM と表示されます)。

    レジストリへの操作を非表示にするにはツールバーの Show Registry Activity ボタンの選択を解除し、レジストリへの操作を再表示するにはそのボタンを押します。

  • Process
    Process Monitor はプロセス/スレッド監視サブシステムで、すべてのプロセスとスレッドの作成および終了、そして DLL とデバイスドライバの読み込みを監視します。

    プロセスへの操作を非表示にするにはツールバーの Show Process and Thread Activity ボタンの選択を解除し、プロセスへの操作を再表示するにはそのボタンを押します。

  • Network
    Process Monitor は Event Tracing for Windows (ETW) を使用して、TCP および UDP のアクティビティを追跡し記録します。各ネットワーク操作には、送信元アドレスと宛先アドレス、および送信または受信データ量を含みますが、実際のデータは含みません。

    ネットワークへの操作を非表示にするにはツールバーの Show Network Activity ボタンの選択を解除し、ネットワークへの操作を再表示するにはそのボタンを押します。

  • Profiling
    このイベントクラスは Options メニューから有効化できます。有効化すると、Process Monitor はシステム内のすべてのアクティブなスレッドをスキャンし、前回のプロファイルイベント以降にスレッドによって使用されたカーネルモードおよびユーザーモードの CPU 時間、コンテキストスイッチの数を記録するプロファイルイベントを生成します。注意: System プロセスはプロファイリングの対象に含まれません。

Process Monitor の基本的な動作を制御するために、いくつかのオプションがあります。

Capture: File メニューの Capture Events エントリ、ツールバーの Capture ボタン、または Ctrl+E ホットキーを使用して、監視するかどうかを切り替えます。

Autoscroll: Edit メニューの Auto Scroll エントリ、ツールバーの Autoscroll ボタン、または Ctrl+A ホットキーを使用して、自動スクロールさせるかどうかを切り替えます。これにより最新の動作が確実に表示されます。

Clear: すべての表示を消去するには、Edit メニューから Clear Display を選択するか Ctrl+X ホットキーを使用します。
