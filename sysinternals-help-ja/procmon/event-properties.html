Process Monitor 日本語解説 - イベントのプロパティ

Sysinternals 日本語解説 > Process Monitor > イベントのプロパティ

個々のイベントのプロパティにアクセスするには、イベントをダブルクリックするか、Event メニューかイベントを右クリックした時のコンテキストメニューから Properties メニュー項目を選択します。Event Properties ダイアログは Event、Process そして Stack ページで構成されています。Event Properties ダイアログの下部にある矢印ボタンで、次か前に表示または強調表示されたイベントに移動することができます。

Event

Event ページにはそのイベントに固有の情報が表示されます。これにはイベントを発生させたスレッド、イベントクラスおよび操作の種類、結果コード、タイムスタンプ、そして適用できるならリソースのパスなどが含まれます。リソースパスは File System と Registry イベントのみで定義されます。Event ページの下部には、そのイベント操作に付随する収集された詳細情報が表示されます。この詳細情報はメイン画面の Detail 列と同じものですが、こちらではそれぞれの情報が複数行で表示されます。

Process

Process ページにはイベントを実行したプロセスに関する情報が表示されます。パスやバージョン文字列のようなプロセスのイメージに関連するデータに加えて、プロセス ID、プロセスを実行しているユーザーアカウント、そしてイベントが 64 ビット Windows システム上で生成された場合はプロセスが 32 ビットか 64 ビットか、といったプロセスの実行属性を表示します。プロセスの整合性レベルや仮想化されているかどうかも表示します。

Process ページの下部には、イベントが実行された時点でプロセス内に読み込まれていたイメージの一覧およびアドレスが表示されます。リスト内のイメージをダブルクリックすると、バージョン情報を含むイメージに関する詳細情報が表示されます。

Stack

Stack ページにはイベントが記録されたときのスレッドのスレッドスタックが表示されます。スタックは、そのイベントを発生させたコンポーネントや発生した理由を調べるのに役立ちます。スタックのカーネルモードフレームには文字「K」が、ユーザーモードスタックには文字「U」が、そのフレームの左側に表示されます。Process Monitor は、トレースで参照されているイメージのシンボルが利用可能なら、アドレスを関数名に変換しようとします。Microsoft シンボルサーバーなどネットワークからシンボルを取得する必要がある場合には、シンボルの解決に時間がかかることがあります。シンボルを設定するには、Options メニューから利用できる Configure Symbols ダイアログを使用します。

Configure Symbols ダイアログでソースファイルのパスを指定すると、行番号シンボル情報が利用可能で、指定したパスにソースファイルが存在していれば Stack ダイアログの Source ボタンが有効になります。Source ボタンをクリックするとテキストビューアが開き、参照されているソースコード行が強調表示されます。

スタックトレースに一覧表示されているイメージの詳細を表示するには、そのフレームをダブルクリックするか、そのフレームを選択してスタックトレース領域の下にある Properties ボタンを押します。

Event メニューの Stack メニュー項目を選択すると、Event Properties ダイアログボックスの Stack ページが直接開きます。
