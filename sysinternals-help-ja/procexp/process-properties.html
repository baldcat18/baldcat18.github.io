Process Explorer 日本語解説 - プロセスのプロパティ

Sysinternals 日本語解説 > Process Explorer > プロセスのプロパティ

プロセスの追加の詳細を表示するにはそのプロセスをダブルクリックするか、プロセスを選択して Process|Properties メニュー項目またはツールバーの [Properties] ボタンを使用します。ダイアログにはいくつかのタブがあります (後述)。パフォーマンス情報などの動的データは、Process Explorer で現在設定されている間隔で更新されます。F5 を押すと動的な情報を手動で更新できます。

Image:

このページには、プロセスのイメージファイルから取得したバージョン情報、イメージファイルのフルパス、プロセスを起動した時のコマンドラインが表示されます。また、プロセスのカレント ディレクトリ、プロセスを実行しているユーザーアカウント、親プロセスの名前、およびプロセスの起動時刻も表示されます。

Process Explorer は、コンピュータが信頼する証明書ルート機関によってイメージがデジタル署名されているかどうかをチェックし、「Verified」(署名済み)、「Unable to verify」、または「Not Verified」(署名が確認されていない) のいずれかの状態を表示します。[Verify] ボタンを押すとまだ検証されていないイメージの署名を Process Explorer が確認します。検証のために証明書の有効性をチェックするのに、Process Explorer は Web サイトに接続します。Verify Image Signatures オプションを参照してください。

Comment フィールドにプロセスのコメントを入力すると、そのコメントがプロセスビューの Comment 列に表示されます。Comment 列が選択されていない場合はプロセス上にマウスを置いたときに表示されるツールチップに表示されます。コメントは同じパスを持つすべてのプロセスに適用されます。

Data Execution Protection (DEP) をサポートするシステムでは、Process Explorer は選択したプロセスの DEP の状態を「Enabled」または「Disabled」として表示します。ソフトウェア DEP は Windows XP SP2 以降の 32 ビット x86 システムでサポートされていますが、ハードウェア DEP は 64 ビットバージョンの Windows でのみ使用できます。プロセスビューに [DEP status] 列を追加することでも DEP の状態を表示できます。

Performance:

このページにはメモリと CPU のパフォーマンスデータが表示されます。これには物理メモリ、仮想メモリ、および CPU 使用率が含まれます。データはメインディスプレイと同じ間隔で更新されます。

Performance Graph:

このページではプロセスのCPU 使用率とプライベートバイト割り当ての履歴をタスク マネージャーのようなグラフで表示します。CPU 使用率グラフの赤色はカーネルモードの CPU 使用率を、緑色はカーネルモードとユーザーモードでの実行の合計を表しています。プライベートバイトは、プロセスが割り当てたプライベート仮想メモリの量を表し、プロセスにメモリリーク バグが発生しているのを示すことがある値です。Process Explorer が通知領域に最小化されている間システム情報パフォーマンスグラフは更新されますが、これらのグラフは更新されません。プライベートバイト使用量グラフは、プロセスが割り当てたプライベートバイトの最大値に対してスケーリングされます。最大値が大きくなるとグラフのスケールが再計算されます。I/O グラフの青色の線はプロセスの読み取りと書き込みの I/O トラフィック量の合計を、ピンクの線は書き込みトラフィック量を表します。I/O グラフは監視を開始してからプロセスが生成した最大 I/O トラフィック量に対してスケーリングされます。

グラフの上でマウスを移動させると、対応する時点でのデータがポップアップで表示されます。

Threads:

このタブにはプロセスで実行中のスレッドのリストが表示されます。スレッドリストには Windows のシンボルエンジンによって提供される開始アドレス情報が表示されます。開始アドレスの正確な名前を確認するには、構成しているシンボルの指示に従ってください。

Threads ページにある Module ボタンを押すと、現在選択されているスレッドの開始アドレスを含むイメージファイルの情報を、エクスプローラーのファイルプロパティ ダイアログボックスで表示します。Stack ボタンは、選択されているスレッドの現在のスタックを表示します。スタック情報はスタック内で参照されているプロセスや DLL 用のシンボルファイルが使用できる場合を除き信頼性がありません。

Kill ボタンを押すとスレッドが強制終了されます。スレッドを終了するとプロセスがクラッシュしたり異常動作したりする可能性があることに注意してください。

Suspend ボタンを押すとスレッドが一時停止されます。スレッドの一時停止はプロセスの実行が停止する原因になる場合があることに注意してください。

TCP/IP:

このページにはプロセスが所有するアクティブな TCP や UDP のエンドポイントが表示されます。

Security:

Process Explorer はこのページで、プロセスのグループとセキュリティトークンに載っている権限の一覧を報告します。[Permissions] ボタンはプロセスに割り当てられたアクセス許可を表示する Permissions エディターを開きます。

Job:

このタブは Win32 ジョブの一部であるプロセスにのみ表示されます。Job ページは同じジョブに関連付けされているプロセスのリストとジョブに適用されている制限を表示します。

.NET Assemblies:

このタブは Process Explorer が管理者権限で実行されている時に、.NET Framework を使用するマネージ プロセスでのみ表示されます。AppDomain ごとに読み込まれたアセンブリをツリービューで表示します。

.NET Performance:

このタブは .NET Framework を使用するマネージ プロセスでのみ表示されます。このプロセス内の AppDomains と、使用可能な .NET パフォーマンス カウンターオブジェクトを表示します。[.NET performance Object] を選択するとそのオブジェクトのカウンター値が表示されます。カウンターは現在設定されている表示更新間隔で更新され、F5 キーを押すことで手動更新できます。

Services:

このタブは Win32 サービスを実行しているプロセスでのみ表示され、プロセス内で実行されているサービスを一覧表示します。Process Explorer はサービスの内部名と表示名、利用可能ならサービスの説明を表示します。[Permissions] ボタンはサービスに割り当てられたアクセス許可を表す Permissions エディターを開きます。

Environment:

このページにはプロセスの環境変数が表示されます。

Strings:

このページには 3 文字以上の印字可能文字列がすべて表示されます。Image 文字列はディスク上にあるプロセスのイメージファイルから読み込まれ、Memory 文字列はイメージのメモリ領域から読み込まれます。イメージがメモリに読み込まれる時に圧縮や暗号化が解除される場合、メモリ内の文字列はディスク上の文字列と異なる事があります。
