Process Explorer 日本語解説 - プロセスのコンテキスト メニュー

Sysinternals 日本語解説 > Process Explorer > プロセスのコンテキスト メニュー

プロセスを右クリックすることで Process メニューと同じ項目にアクセスできます。これらの項目を使用すると次の操作を実行できます。

Bring to Front: このオプションを選択すると、そのプロセスが所有するウィンドウが最前面に表示されます。

Set Priority: このサブメニューでプロセスの基本優先度を変更できます。プロセスの基本優先度を変更すると、システムは新しい基本優先度と同じ相対優先度を維持することでそのプロセス内のスレッドの優先度を調整します。

Set Affinity: 複数の CPU を持つシステムでは、このメニュー項目を使用してプロセスのスレッドを実行する CPU を特定の CPU に限定できます。

Debug: このメニュー項目を選択すると、HKEY_LOCAL_MACHINE\Software\Microsoft\Windows NT\CurrentVersion\AeDebug に登録されているデバッガを選択したプロセスをコマンドライン引数に設定して起動します。

Launch Depends: Process Explorer が Dependency Walker ツール (http://www.dependencywalker.com を参照) を見つけると、選択したプロセスを引数として起動します。Dependency Walker ツールは DLL の静的な依存関係を表示します。

Kill Process: この項目は TerminateProcess API を使用してプロセスを強制終了させます。この方法で終了したプロセスは終了について警告しないため、保存されていないデータは書き込まれません。

Kill Process Tree: このメニュー項目はプロセスペインがプロセスツリーモードである場合のみ使用可能です。プロセスとその子孫プロセスすべてを強制終了できます。

Suspend: ネットワーク、CPU、ディスクなどのシステムリソースを他のプロセスで使用できるようにするためにプロセスを一時的に非アクティブにしたい場合に、そのプロセスを一時停止できます。中断したプロセスは灰色で表示されます。中断したプロセスを再開するにはプロセスのコンテキストメニューから [Resume] を選択します。

Restart: この項目を選択すると、Process Explorer は強調表示されたプロセスを終了し、同じイメージを同じコマンドライン引数を使用して起動します。元のプロセスが別のユーザーアカウントや異なる環境で実行されていた場合、新しいインスタンスの起動に失敗したり異なる動作をする可能性があることに注意してください。

Properties: この項目を選択すると、プロセスに関する詳細情報を表示するプロパティ ダイアログを開きます。

Search Online: この項目を選択すると、Process Explorer はシステムの既定のインターネットブラウザーを起動し、選択したプロセスの名前でインターネット検索を開始します。
