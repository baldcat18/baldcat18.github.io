Process Explorer 日本語解説 - システム情報

Sysinternals 日本語解説 > Process Explorer > システム情報

View メニューにある System Information 項目を選ぶか Ctrl+I を押すと、タスク マネージャーにあるようなシステム全体のパフォーマンス統計を表示するダイアログ ボックスが開きます。この情報には、コミットされた仮想メモリと物理メモリ、およびページプールや非ページプールに割り当てられたカーネル バッファーのサイズが含まれます。

グラフには、システムの CPU 使用率だけでなく、コミットされた仮想メモリのサイズ、I/O 使用量、GPU 使用率の履歴が表示されます。CPU 使用率グラフの赤色はカーネルモードの CPU 使用率を表し、緑色はカーネルモードとユーザーモードの実行の合計を表します。コミットされた仮想メモリがシステムのコミット制限に達すると、アプリケーションとシステムが不安定になります。コミット制限は、物理メモリとページングファイルのサイズの合計とほぼ同じです。I/O グラフの青い線は I/O トラフィックの合計を表します。これは、すべてのプロセスの読み取りと書き込み量を足したものです。ピンクの線は、前回の更新以降の書き込みトラフィックを表します。

CPU グラフ上でマウスを動かすとポップアップが表示され、対応する時点での CPU 使用率と、CPU 使用率が最も高かったプロセスの CPU 使用率と名前が時刻とともに表示されます。同様にコミット グラフでも、その時点での情報が表示されます。最後に、I/O グラフのツールチップにはその時点で最も I/O を実行するプロセス名が、読み書きしたデータの量とともに表示されます。ポップアップに表示される情報はグラフがマウスの下を移動するにつれて更新されますが、マウスを左クリックすることで更新が一時停止されます。マウスを動かすと一時停止が解除されます。

複数の CPU を持つシステムでは、[System Information] ダイアログに Show one graph per CPU チェックボックスが表示されます。これをチェックすると、グラフがプロセッサーごとの表示に切り替わります。同じコアを共有するハイパースレッディング (SMT) プロセッサーと、同じノードを共有する NUMA プロセッサーはグループ化され、グラフの上でマウスを移動すると表示されるツールチップには、プロセッサー番号とともにコア番号やノード番号を表示します。CPU グラフのツールチップには、特定のCPU 上で最も CPU を消費したプロセスではなく、関連付けられた時間にシステム全体で最も CPU を消費したプロセスの名前が表示されることに注意してください。
