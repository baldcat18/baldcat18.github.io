Autoruns 日本語解説 - 表示する場所とエントリ

Sysinternals 日本語解説 > Autoruns > 表示する場所とエントリ

ただ Autoruns を実行するだけで、現在アプリケーションを最も直接的に実行する場所で自動起動するよう設定されているアプリケーションが表示されます。表示を更新するとオプションの変更を反映した新しいスキャンを実行します。

  • Logon このエントリは、現在のユーザーとすべてのユーザーのスタートアップ フォルダー、Run レジストリキー、そして標準のアプリケーション起動場所のような、標準的な自動起動エントリの場所をスキャンします。
  • Explorer このエントリを選択すると、エクスプローラーのシェル拡張、ブラウザーヘルパーオブジェクト、エクスプローラーのツールバー、active setup の実行、および shell execute hook を表示します。
  • Internet Explorer このエントリにはブラウザーヘルパーオブジェクト (BHO)、Internet Explorer のツールバーと拡張機能が表示されます。
  • Services すべての Windows サービスはシステムの起動時に自動的に開始するように設定されています。
  • Drivers ここにはシステムに登録されているカーネルモードドライバーが、無効になっているものを除いてすべて表示されます。
  • Scheduled Tasks 起動時またはログオン時に開始するように構成されたタスク スケジューラのタスク。
  • AppInit ここにはアプリケーション初期化 DLL として登録されたDLLが表示されます。
  • Boot Execute 起動プロセスの初期段階で実行される (Windows イメージではない) ネイティブイメージ。
  • Image Hijacks Image File Execution Options とコマンドプロンプト起動時の自動開始エントリ。
  • KnownDLLs これは Windows がロードするアプリケーションによって参照される DLL の場所を報告します。
  • Winsock Providers Winsock サービスプロバイダーを含む登録済みの Winsock プロトコルを表示します。マルウェアは自身を Winsock サービスプロバイダーとしてインストールすることがよくあります。Winsock サービスプロバイダーを削除するツールがほとんど無いからです。
  • LSA Providers 登録されているローカルセキュリティ機関 (LSA) の認証、通知、およびセキュリティパッケージを表示します。
  • Print Monitors print spooling サービスによってロードされるDLLを表示します。マルウェアは自身を自動起動させるためにこのサポートを使用していたことがありました。
  • Sidebar Gadgets Windows サイドバーガジェットを表示します。

Options メニューの Hide Empty Locations が有効化されている時、Autoruns はエントリがない場所を表示しません。

User メニューにはユーザー名が入力されます。その中の 1 つを選択すると、そのアカウント用の自動開始イメージが表示されます。
